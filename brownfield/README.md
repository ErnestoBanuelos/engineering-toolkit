# Brownfield

## Purpose

The Brownfield domain contains practices and assets for engineering activities on existing and legacy systems.

Brownfield work is a first-class engineering scenario in the Engineering Toolkit.

---

## Responsibilities

- Provide procedures for reverse engineering and understanding existing systems.
- Define Delta Specification practices for capturing intended changes.
- Support safe migration and modernization planning.
- Enable characterization testing before modification.

---

## What Belongs Here

- Reverse engineering procedures and checklists.
- Delta Specification methodology.
- Legacy system assessment frameworks.
- Migration planning assets.
- Characterization testing procedures.
- Brownfield-specific Playbooks and Workflows.

---

## What Does NOT Belong Here

- Greenfield-only practices (those belong to `skills/` or `playbooks/`).
- Project-specific legacy system documentation (that belongs to the project).
- Implementation code.

---

## Brownfield is a Support Domain

Brownfield assets depend on Skills and Templates.

They apply these capabilities to the specific challenges of existing systems.
