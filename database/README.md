# Database

## Purpose

The Database domain contains engineering assets that address the specific risks and practices of database engineering.

---

## Responsibilities

- Provide SQL review procedures.
- Define migration review standards and checklists.
- Support schema analysis and performance review.
- Capture database-specific engineering knowledge.

---

## What Belongs Here

- SQL review procedures and checklists.
- Database migration review assets.
- Schema analysis procedures.
- Database performance review guidelines.
- Database-specific Standards and Templates.

---

## What Does NOT Belong Here

- Project-specific database schemas or migrations (those belong to the project).
- Generic engineering Standards not specific to databases (those belong to `standards/`).
- ORM or framework-specific implementation code.

---

## Database is a Support Domain

Database assets depend on Skills and Templates.

They apply these capabilities to the specific domain of database engineering.
