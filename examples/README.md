# Examples

## Purpose

The Examples domain contains reference implementations that demonstrate recommended usage of Engineering Toolkit assets.

---

## Responsibilities

- Demonstrate how Skills, Templates, Workflows, and Playbooks are used in practice.
- Provide sample Context Bundle structures (structure only — no project content).
- Illustrate Skill composition patterns.
- Support onboarding through concrete worked examples.

---

## What Belongs Here

- Sample Skills demonstrating the authoring standard.
- Sample Specifications illustrating template usage.
- Example Context Bundle structures (empty, with placeholders only).
- Illustrated Workflow compositions.
- Onboarding walkthroughs.

---

## What Does NOT Belong Here

- Normative assets (examples are educational, not authoritative).
- Real project content or customer information.
- Actual Context Bundles with project data.
- Production-ready assets (those belong to their respective domains).

---

## Critical Constraint

**Examples are educational. They are not normative.**

An example must never be referenced as the authoritative source for a capability.

If an example demonstrates a valuable practice, that practice should be extracted into a proper Engineering Asset in the appropriate domain.

Examples should reference the normative assets they demonstrate, not the other way around.
